# 圆柱孔布尔运算模块改进技术文档

## 摘要

本文档记录了三维网格编辑系统中圆柱孔布尔运算模块（CSG Hole）的关键改进内容，涵盖定向圆柱生成算法的重构、通孔与盲孔的自适应射线检测机制，以及布尔运算鲁棒性的提升。改进后的方案有效解决了旧版本在深度控制、方向对齐和布尔运算共面失败等方面的缺陷，为基于网格的交互式三维建模提供了更可靠的底层支撑。

---

## 1. 背景与问题分析

### 1.1 功能概述

CSG 圆柱孔模块负责在已有三维网格模型上执行圆柱形减法布尔运算（difference），实现"打孔"操作。其核心流程为：

1. 根据用户拾取的表面点（`center`）和法线方向（`normal`）生成工具圆柱；
2. 将工具圆柱与主体网格执行差集布尔运算，得到带孔网格；
3. 向前端返回更新后的网格数据供可视化渲染。

### 1.2 旧版本缺陷

旧版本 `_make_oriented_cylinder` 函数及 `csg_hole` 接口存在以下问题：

| 问题类型 | 具体表现 |
|---|---|
| **深度控制不精确** | 圆柱以轴向中点为 center，深度仅靠 `depth * 1.05` 固定系数补偿，既无法适应通孔，也难以保证盲孔精度 |
| **方向处理不完整** | 旧版本未处理法线与 Z 轴完全反向（dot ≈ −1）的退化情形，叉积趋零导致旋转轴无效 |
| **布尔运算不稳定** | 圆柱端面与模型表面共面时，CGAL 布尔运算会产生退化结果 |
| **通孔判断缺失** | 无法区分通孔与盲孔，深度不足时通孔无法被完整切穿 |

## 2. 改进方案

### 2.1 定向圆柱生成算法重构

#### 2.1.1 坐标系与方向约定

改进后明确了方向约定：

- **+normal 方向**：出射侧（摄像头方向，模型外部）
- **−normal 方向**：入射侧（钻入方向，模型内部）
- **center**：用户拾取的表面入口点

圆柱的空间范围由两个独立参数控制：

```
底面位置 = center − normal × outward_ext   （朝出射侧延伸）
顶面位置 = center + normal × inward_ext    （朝入射侧延伸）
总高度   = inward_ext + outward_ext
```

#### 2.1.2 圆柱生成流程

**步骤一：在局部坐标系生成圆柱**

调用底层接口 `HGP_Mesh_Make_Cylinder(cx, cy, cz, r, h, seg)`，其中 `cz` 为轴向几何中心。令：

```
cz = total_height / 2
```

使圆柱底面落在原点（z = 0），顶面在 z = total_height，便于后续绕原点旋转时底面圆心保持固定。

**步骤二：绕原点旋转对齐法线方向**

计算 Z 轴与目标法线之间的旋转变换：

```
dot = Z · normal
旋转轴 = Z × normal（归一化）
旋转角 = arccos(dot)
```

针对旧版本遗漏的退化情形，增加如下处理：

- 若 `dot ≈ +1`（法线已与 Z 轴对齐），跳过旋转；
- 若 `dot ≈ −1`（法线与 Z 轴完全反向，叉积为零向量），绕 X 轴旋转 π 角，确保旋转轴合法。

旋转通过调用 `HGP_Rotation_Obj` 在文件层面完成，绕原点执行，底面圆心在旋转全程保持在原点不动。

**步骤三：平移到目标位置**

旋转完成后，将底面圆心从原点平移至：

```
origin_after_translation = center − normal × outward_ext
```

即：

```python
ox = center[0] - ax * outward_ext
oy = center[1] - ay * outward_ext
oz = center[2] - az * outward_ext
cyl_verts = [[v[0]+ox, v[1]+oy, v[2]+oz] for v in cyl_verts]
```

平移后，底面位于入口点的出射侧，顶面深入模型内部，满足布尔运算的几何要求。

#### 2.1.3 新旧接口对比

| 参数 | 旧版本 | 新版本 |
|---|---|---|
| 高度参数 | `height`（单一深度） | `inward_ext` + `outward_ext`（双向独立控制） |
| 中心语义 | 轴向几何中心 | 入口拾取点（表面点） |
| 反向退化处理 | 无 | 绕 X 轴旋转 π |
| 调用约定 | 隐式 | 显式文档化 |

### 2.2 通孔与盲孔自适应检测

#### 2.2.1 问题动机

对于通孔（圆柱完全穿透模型），若仅按用户输入的 `depth` 参数生成工具圆柱，当 `depth` 等于或略大于模型厚度时，圆柱端面可能与模型对面共面，导致布尔运算结果异常或失败。

#### 2.2.2 射线检测机制

在执行布尔运算前，沿 −normal 方向发射一条射线，检测入口点对侧的出口点：

1. **偏移起点**：将射线起点沿 −normal 方向偏移 `EPS_OFFSET`（取 `max(diag × 0.005, 1e-4)`），避免自交：

```python
shifted = center − normal × EPS_OFFSET
ray_dir = −normal
```

2. **射线-网格求交**：调用 `HGP_3D_Intersection_Rays_Mesh_Vector3d`，基于 `.off` 格式的 Surface Mesh 执行精确求交；

3. **计算模型厚度**：

```python
dist      = |hit_point − shifted|
thickness = EPS_OFFSET + dist
```

#### 2.2.3 通孔/盲孔判定逻辑

```
if thickness 有效 and depth >= thickness:
    → 通孔：inward_ext = depth + small_eps   （保证完全穿透，small_eps 防共面）
else:
    → 盲孔：inward_ext = depth               （按用户深度精确钻入）
```

其中 `small_eps = min(diag × 0.001, thickness × 0.05)`，为自适应微量偏移，仅用于消除布尔运算共面退化，不影响用户感知深度。

`outward_ext` 统一设为包围盒对角线长度 `diag`，确保出射侧圆柱始终超出模型边界，布尔运算完整切除入口面材料。

#### 2.2.4 包围盒对角线的作用

使用包围盒对角线 `diag` 作为"足够大"的无穷延伸量，满足：

- 出射侧圆柱必然超出模型任意方向的边界；
- 通孔场景下圆柱端面不与模型对面共面；
- 避免引入数值过大的全局常量，保持与模型尺度自适应。

---

### 2.3 预览圆柱与打孔圆柱的一致性

改进前，预览接口 `csg_preview_cylinder` 使用与打孔接口不同的参数生成圆柱，导致前端预览形状与实际打孔结果存在偏差。

改进后，预览圆柱的几何形状与打孔圆柱的入射侧完全一致：

```python
# 预览接口
_make_oriented_cylinder(
    center, ray_dir, radius,
    inward_ext  = depth,   # 与打孔一致：用户设定深度
    outward_ext = 0.0      # 预览不显示出射侧超出部分
)
```

这样前端所见即所得，用户预览时看到的圆柱深度与最终打孔深度完全吻合。

## 3. 算法正确性分析

### 3.1 坐标推导验证

设 `normal = (ax, ay, az)`，`center = (cx, cy, cz)`，以下验证各关键点坐标。

**局部坐标系（旋转前）：**
- 底面圆心：`(0, 0, 0)`
- 顶面圆心：`(0, 0, total_height)`

**旋转后（绕原点，底面圆心不动）：**
- 底面圆心：`(0, 0, 0)`
- 顶面圆心：`normal × total_height`

**平移后（底面圆心 → `center − normal × outward_ext`）：**
- 底面圆心：`center − normal × outward_ext`
- 顶面圆心：`center − normal × outward_ext + normal × total_height`
           `= center + normal × (total_height − outward_ext)`
           `= center + normal × inward_ext` ✓

结论：底面位于 center 的出射侧（距离 outward_ext），顶面位于 center 的入射侧（距离 inward_ext），与设计预期完全吻合。

### 3.2 退化情形鲁棒性

| 情形 | 旧版本 | 新版本 |
|---|---|---|
| `normal ≈ +Z` | 正常（跳过旋转）| 正常（跳过旋转） |
| `normal ≈ −Z` | 叉积为零，旋转轴无效，结果错误 | 绕 X 轴旋转 π，结果正确 |
| 射线未命中对面 | 无此判断，按固定系数延伸 | 按盲孔处理，`inward_ext = depth` |
| 深度恰好等于厚度 | 端面共面，布尔失败 | 加 `small_eps` 偏移，布尔成功 |

## 4. 接口规范

### 4.1 `_make_oriented_cylinder`

```python
def _make_oriented_cylinder(
    center,             # [x, y, z]，拾取点（表面入口）
    normal_normalized,  # [ax, ay, az]，单位法线，+方向为出射侧
    radius,             # float，圆柱半径
    inward_ext,         # float，朝 -normal 方向（模型内部）延伸量
    outward_ext,        # float，朝 +normal 方向（模型外部）延伸量
    segments=48         # int，圆周细分数
) -> (verts, fi0, fi1, fi2)
```

**调用约定：**

| 调用场景 | `inward_ext` | `outward_ext` |
|---|---|---|
| `csg_hole`（打孔） | `depth`（盲孔）或 `depth + small_eps`（通孔） | `diag`（包围盒对角线） |
| `csg_preview_cylinder`（预览） | `depth` | `0.0` |

### 4.2 `POST /api/csg/hole`

**请求体（`CSGHoleRequest`）：**

| 字段 | 类型 | 说明 |
|---|---|---|
| `mesh_id` | str | 主体网格 ID |
| `center` | [float×3] | 拾取点坐标 |
| `normal` | [float×3] | 表面法线（无需单位化） |
| `radius` | float | 孔半径 |
| `depth` | float | 打孔深度 |

**处理流程：**
1. 计算包围盒对角线 `diag`；
2. 射线检测判定通孔/盲孔；
3. 调用 `_make_oriented_cylinder` 生成工具圆柱（`ray_dir = −normal`）；
4. 执行 difference 布尔运算；
5. 保存并返回结果网格。

### 4.3 `POST /api/csg/preview_cylinder`

**处理流程：**
1. 生成 `inward_ext = depth`、`outward_ext = 0` 的预览圆柱；
2. 保存并返回圆柱网格（不修改主体网格）。

## 5. 与旧版本的差异汇总

| 维度 | 旧版本 | 新版本 |
|---|---|---|
| 圆柱中心语义 | 轴向几何中心 | 表面拾取点（入口） |
| 深度参数 | 单一 `height`，固定 ×1.05 补偿 | `inward_ext` + `outward_ext` 双向独立 |
| 通孔支持 | 无 | 射线检测自动判定并延伸 |
| 反向法线退化 | 未处理，结果错误 | 绕 X 轴旋转 π |
| 共面防护 | 固定 ×1.05，可能不足 | `small_eps` 自适应偏移 |
| 预览一致性 | 与打孔参数不同步 | 入射侧几何完全一致 |
| 出射侧延伸 | 固定比例 | `diag`（自适应模型尺度） |

---

## 6. 结论

本次改进从算法设计、接口规范和鲁棒性三个层面对圆柱孔布尔运算模块进行了系统性重构：

1. **几何精确性**：通过明确的双向延伸参数和严格的坐标推导，圆柱的空间位置与用户意图完全对应；
2. **鲁棒性**：处理了法线反向退化、射线未命中、端面共面三类边界情形，消除了旧版本的潜在崩溃或错误结果；
3. **自适应性**：利用包围盒对角线和自适应微量偏移，算法行为与模型尺度无关，无需人工调参；
4. **一致性**：预览圆柱与实际打孔圆柱几何一致，提升了交互体验的可预期性。

上述改进已集成于 `vis/backend/app.py` 中的 `_make_oriented_cylinder`、`csg_hole` 和 `csg_preview_cylinder` 函数。
