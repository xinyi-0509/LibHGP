## 2. 每页可直接使用的“标题 + 3点正文”

### P2 研究背景与问题定义
- Python 工具（如 trimesh、Open3D）上手快，但复杂任务稳定性与一致性受限。  
- 可视化框架（Three.js、FastAPI）便于展示，但常停留在脚本/展示层。  
- 缺少贯通“几何内核—跨语言桥接—交互验证—制造应用”的统一引擎。  

### P3 总体目标与贡献
- 复用实验室长期积累的 LibLGP / LibHGP 算法资产。  
- 建立 C++/Python/Web 多层协同的工程化交付链路。  
- 以 CSG 打孔任务验证系统可用性与工程可靠性。  

### P4 系统分层架构
- **算法层**：核心几何计算能力（`hgp2d.cpp`、`hgp3d.cpp`、`hgpmesh.cpp` 等）。  
- **接口层**：统一 C++ API 与 Python 绑定（`libhgp.h`、`hgp_py.cpp`）。  
- **平台/应用层**：`vis/backend/app.py` 与 CSG 交互流程。  

### P5 底层工程化整合（C++）
- 通过 `CMakeLists.txt` 统一构建动态库与静态库。  
- 统一导出边界，降低上层调用耦合成本。  
- 为 Python 模块静态链接提供稳定底座。  

### P6 Python 扩展层
- `to_vec2d/to_vec3d` 等转换函数负责类型映射与边界检查。  
- 非法输入在边界处抛错，错误可直接反馈到 Python 调用侧。  
- Lambda 包装器将多输出 C++ 接口重组为 Python 友好返回值。  

### P7 可视化平台
- 本地化部署，支持上传、缓存、算法调用、结果回显。  
- 服务端调度与前端显示解耦，便于功能扩展。  
- 可用于算法效果复核与交互式演示。  

### P8 应用落地：CSG 打孔
- 构造“标准圆柱—方向对齐—空间平移”的定向工具体。  
- 结合局部厚度估计区分通孔与盲孔语义。  
- 形成“预览—确认—执行—回退”的闭环。  

### P9 实验与验证
- 分层验证：库加载、Python 调用、平台流程、场景应用。  
- 展示 1~2 组关键输入/输出案例。  
- 强调“可运行、可复现、可扩展”。  

### P10 创新点总结
- 工程化整合：从研究代码到可交付组件。  
- 跨语言桥接：系统化类型转换与返回值重组。  
- 场景闭环验证：制造语义驱动的 CSG 打孔落地。
